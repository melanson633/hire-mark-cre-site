What I Built

  • Claude Code / Codex Skill (build-1040-estimate-workbook) that orchestrates subagents across client tax forms, consolidates extracted values, populates the template, and records unresolved items — modular and reusable across any client file mix.
  • Locked-in 6-sheet Excel template: Estimate, SchC, SchD, SchE, Rates, and Summary. Live formulas (nothing hardcoded), manual-override cells for ambiguous items, and embedded TY2025 constants.
  • Tie-out discipline per populated field: field name, workbook amount, source filename, source amount, status (MATCH / MISMATCH / UNVERIFIED / MANUAL), and a short note — so every number is traceable back to its source.
  • Advisor summary & analytics on-page: refund-or-balance-due, safe-harbor gap, effective marginal rate (ordinary vs. LTCG), and an explicit Unresolved / Follow-Up Items list.

Implementation — Inputs, Logic, Controls

Inputs

W-2s, 1099-INT / DIV / MISC / NEC / K, K-1s, 1098 mortgage, prior-year return, and pay stubs (used by a W-2 sidebar when year-end wages must be estimated). User-entered assumptions fill manual-override cells when a source is ambiguous or missing.

Processing Logic

The Skill spawns subagents to parallelize document extraction by form type. Extracted values are consolidated before the workbook is touched, then the template is populated only in designated manual-input cells. Schedules C, D, and E are populated only when the client actually has those items. Live formulas on Estimate, Summary, and Rates do all computation — nothing is ever hardcoded.

Outputs & Controls

A 6-sheet workbook with a refund / balance-due headline, safe-harbor analysis, marginal-rate analysis, and an explicit unresolved-items list. Every populated field is tie-out-validated (MATCH / MISMATCH / UNVERIFIED / MANUAL); the preparer signs off before any number is shared with a client.

AI in the Build

AI is used as leverage at three specific seams. Durable calculation logic sits in live Excel formulas and a locked-in template; AI accelerates authoring, extraction, and review.

Qualitative Impact

  • Cleaner review: every field carries a source filename and a tie-out status — defensible numbers, not guessed numbers.
  • Faster issue surfacing: the on-sheet Unresolved / Follow-Up list drives the next client email automatically.
  • Scenario speed: Sch C / Sch E changes reflow live through Estimate and Summary — no edit-and-revert cycles inside tax software.
  • Consistent client communication: every client gets the same structured outputs (refund / tax due, safe-harbor gap, marginal rate, open items).

Transferable Relevance

  • AI fluency: authored a reusable Skill that produces consistent results regardless of input mix — prompt & context engineering applied to a messy, document-heavy workflow.
  • Reusable Excel template discipline: locked-in constants, live formulas, and manual-override cells turn a spreadsheet into a dynamic, easily-updatable model — not a one-off worksheet.
  • Transferability: the same framework (subagent extraction → template population → tie-out validation) applies to any spreadsheet-dominant internal workflow where raw documents must land in a structured model.
  • Implementation judgment: chose the least-fragile path per client (local structured extraction over raw multi-file LLM ingestion), kept calculations live, and gated delivery on preparer signoff.

Limitations & Controls Acknowledged

  • Federal only for TY2025; state tax logic, amended returns, and electronic filing are explicitly out of scope.
  • Ambiguous items are flagged, not auto-classified — Sch C expenses can fall back to prior-year baselines until current-year is confirmed; Form 2441 and situational credits are only modeled when source documents are present.
  • Preparer reviews and signs off before any number is shared with a client; the Skill produces a reviewable artifact, not a filed return.

Appendix — TY2025 Rates Sheet

The Rates sheet is the single source of constants feeding the rest of the model — standard deductions, ordinary brackets for all four filing statuses, LTCG / QD thresholds, SS wage base, NIIT & Additional-Medicare thresholds, QBI, CTC, safe-harbor factors, and SE-tax inputs.

Swap constants here in the next tax year — the rest of the model stays identical.
