Field-to-GL: An AI-Built Mobile Labor Capture and Intercompany Billing Pipeline

What I Built

  • Mobile web app for labor time capture, engineered end-to-end with Replit Agent using deliberate prompt & context engineering. Structured inputs: property, work type (Maintenance / Construction), detail category (Common Area, Tenant Specific, Vacant Space, or active Project), hours, and free-text notes.
  • Approval layer for property and construction project managers to review weekly entries and submit payroll-ready reports.
  • AI review layer — a custom Shortcut.AI Skill that autonomously flags inconsistent entries using keyword rules and semantic mismatch against entry metadata.
  • ETL pipeline in Power Query / Power Pivot for validation, rate mapping (by role and PMA escalator), charge-type allocation, and last-mile Excel formulas to produce a Yardi-importable CSV.
  • Controls: required fields at capture, weekly PM approvals, monthly AI-assisted review, hour-total reconciliation, and a blended-rate sanity check on total billings.

Transform — Deterministic ETL

Inputs

Date (auto), property, work type, detail category (or active project for construction), hours, billable flag, PTO/holiday, and notes. Project managers maintain the active-project list per property directly in-app.

Processing Logic

Power Query normalizes the app export and joins a rate table (per role; adjustable for role changes and PMA inflation escalators). Power Pivot aggregates by property × charge type (Common Area, Tenant Specific, Vacant Space, Project) and routes to the correct JV entity. Last-mile Excel formulas produce a Yardi import layout: payee, invoice date/#, post month, entity, amount, GL / A/P accounts, notes.

Output

A weekly payroll-submission report reviewed by property and construction PMs; and a monthly Yardi-ready intercompany payable CSV from the operator entity to each JV property entity, with job-cost project and cost-code detail preserved on construction entries.

Qualitative Impact

  • Cleaner billing: standardized charge-type routing and blended-rate sanity checks replaced ad-hoc spreadsheet work.
  • Faster close support: end-to-end monthly billing collapses to under an hour of senior review.
  • Richer entry data: per-entry notes from the field are preserved into the GL package — previously lost in manual re-keying.
  • Better accountability: weekly PM approval and AI-assisted inconsistency flagging sit in front of monthly billing.

Adoption & Continuity

  • Rolled out to the field, property, and construction teams; in continuous monthly use for 10+ months with no reported downtime.
  • Operator ran the monthly pipeline personally; process remained in place at RJK after departure.
  • Reused as both the payroll submission source and the monthly intercompany billing source from a single dataset.

Transferable Relevance

  • End-to-end implementation judgment: field capture, review, ETL, and ERP integration designed as one system — not a point tool bolted on.
  • Business-process understanding: deliberate design of inputs, approvals, reconciliation, and audit-friendly outputs for finance, property, and construction stakeholders.
  • AI / tooling fluency: agentic build tools and custom skills used where they pay; deterministic code where audit integrity matters.
  • Finance-ops relevance: eliminates manual re-keying, tightens close, and feeds ERP postings with structured, audit-defensible data.

Limitations & Controls Acknowledged

  • Monthly human review is required before posting; the AI review layer flags, it does not auto-correct.
  • Rate tables and PMA escalators are maintained manually; changes flow through a single rate source.
  • Yardi posting step is a controlled import — intentionally human-gated, not automated end-to-end.
